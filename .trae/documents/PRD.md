## 1. 产品概述

面向转专业申请者的纯前端作品集诊断工具，帮助用户在本地完成作品集的整理、诊断与导出。无需注册、完全离线、轻量高效，打开即用。

- **核心价值**：帮助转专业申请者系统地梳理作品集，对照目标院校要求诊断能力缺口，确保作品集"讲得通、做得深、证明得了能力"
- **目标用户**：跨专业考研/留学申请者，尤其是从文科、工科转向设计、艺术、交互等专业的学生
- **市场定位**：填补转专业作品集准备过程中缺乏系统化、轻量化诊断工具的空白

## 2. 核心 Features

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|---------|---------|
| 普通用户 | 无需注册，本地使用 | 所有功能，数据本地存储 |

### 2.2 功能模块

1. **创建页**：项目卡片管理、个人背景填写、项目排序
2. **诊断页**：目标专业选择、能力缺口分析、素材完整性检查、页面结构预览
3. **对照页**：提交清单生成、版本对比
4. **导出页**：导出 PDF、离线保存管理

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 创建页 | 项目卡片编辑 | 添加/编辑/删除项目卡片，拖拽排序项目顺序 |
| 创建页 | 背景信息填写 | 填写教育背景、工作经历、技能清单 |
| 创建页 | 项目过程标注 | 标注每个项目从零到一的完整过程（动机→调研→构思→实现→反思） |
| 诊断页 | 目标专业选择 | 选择目标院校和专业，查看招生偏好 |
| 诊断页 | 能力缺口提示 | 基于目标专业要求，分析当前作品集的能力缺口 |
| 诊断页 | 素材完整性检查 | 检查每个项目必备素材是否完整（过程图、成品图、说明文档等） |

| 导出页 | 离线保存 | 本地保存多版本作品集数据，支持导入导出 JSON |

## 3. 核心流程

用户从创建作品集开始，依次填写项目信息和背景资料，选择目标专业进行智能诊断，对照检查清单完善素材，最后导出诊断报告和作品集数据。

## 4. 用户界面设计

### 4.1 设计风格

**美学方向**：学术简约风，融合设计类作品集的精致感与学术工具的专业感

- **主色调**：深靛蓝 `#1e3a5f`（专业、稳重）
- **辅助色**：暖橙 `#f59e0b`（强调、提示）、墨绿 `#059669`（通过、完整）、砖红 `#dc2626`（警告、缺失）
- **中性色**：米白 `#fafaf9` 背景、炭灰 `#1c1917` 文字
- **按钮风格**：微圆角（4px）、扁平化设计、hover 时轻微上浮 + 阴影
- **字体**：标题使用 Noto Serif SC（人文感），正文使用 Noto Sans SC（易读性）
- **布局风格**：卡片式布局、清晰的视觉层次、充足留白
- **图标风格**：Lucide 线性图标，统一 20px 尺寸

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 创建页 | 项目卡片列表 | 可拖拽卡片，包含项目标题、缩略图、进度条、编辑/删除按钮 |
| 创建页 | 项目编辑器 | 模态框表单，分步骤填写（基本信息→过程标注→素材上传） |
| 诊断页 | 专业选择器 | 下拉选择 + 院校/专业标签，右侧展示招生偏好说明 |
| 诊断页 | 能力雷达图 | 六维能力雷达图，对比当前水平与目标要求 |
| 诊断页 | 素材检查列表 | 树形结构，每项带完成状态指示器 |
| 对照页 | 提交清单 | 分组复选列表，显示必填/选填状态 |
| 对照页 | 版本对比 | 双栏布局，高亮差异内容 |
| 导出页 | 导出预览 | PDF 预览缩略图，导出选项配置 |

### 4.3 响应式

- 桌面优先设计（1280px 起步）
- 平板端（768px-1024px）：调整卡片列数为 2 列
- 移动端（<768px）：单列布局，简化交互，隐藏次要功能
- 所有触控目标 ≥ 44px × 44px

### 4.4 动效设计

- 页面切换：淡入淡出 + 轻微位移（200ms ease-out）
- 卡片拖拽：拖拽时轻微放大 + 阴影加深
- 状态变化：完成/通过状态用绿色对勾动画，缺失/警告用红色脉冲提示
- 数据加载：骨架屏脉冲动画
